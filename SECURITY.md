# Security Policy

## Supported Versions

Security updates are provided for the latest released version of `zed-workspace-dock`.

## Reporting a Vulnerability

Do not open a public issue for a suspected vulnerability.

Use GitHub's private vulnerability reporting for this repository once it is public. If private reporting is unavailable, contact the maintainers through the repository owner profile and include:

- The affected version or commit.
- Steps to reproduce.
- Expected impact.
- Any known workaround or mitigation.

Maintainers should acknowledge valid reports, triage the impact, and coordinate a fix through a private security advisory before public disclosure.
